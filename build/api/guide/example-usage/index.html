Usage

We will use code snippets from tests to demonstrate how to use this library.

The core of this package is PaymentGatewayRequester service.

All classes from namespace Helper are just data models required to make API request.

Lifetime cycle

Overview

If we want to construct PaymentGatewayRequester service, we need url link and access token

The PaymentGatewayRequester service has one purpose. Processing requests to payment api and returning responses. Once the service is created, it has only one method PaymentGatewayRequester::makeRequest() . Method can create payment request with provided data in array type (this can be useful, when you want to handle data on your own), or class that implements PaymentGatewayRequest . For standard scenario we have prepared two helper classes, which implements PaymentGatewayRequest .

  • PaymentRequest
  • PaymentInfoRequest

This classes extends AbstractToArray , it means they can cast all properties to array.

PaymentGatewayRequester::makeRequest() method returns CurlResponse object in case no curl error has occurred.

When a curl error has occurred, the CurlRequestException is thrown. It has CurlRequestException::getCurlResponse() method, which retrieves CurlResponse object.

We can check for error message by calling CurlResponse::getErrorMessage() . This method returns null if no error occurred, else error message in string.

If we want to check http response code, we need to call CurlResponse::getStatusCode() .

In case we want to get raw body we will call CurlResponse::getBody() , or its json parsed variant CurlResponse::json() , which returns parsed response data in array.

In this step, we should validate response data.

For this purpose we can use Response data model classes.

  • PaymentResponse - represents data from PaymentRequest
  • PaymentInfoResponse - represents data from PaymentInfoRequest

Both of this classes implements fromArray() method, so we can easy pass data from CurlResponse using CurlResponse::json() method.

PaymentGatewayRequester service

example.php 
<?php

    $paymentGatewayRequester = new PaymentGatewayRequester('https://api_url_from_registration_request', 'token_from_registration_request');
